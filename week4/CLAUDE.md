# Week4 Project Notes

## Project Overview
- FastAPI backend with SQLite database
- Notes and Action Items management API
- Uses SQLAlchemy ORM with dependency injection for database sessions

## Run the app (from `week4/` directory)

```bash
make run
```
Open `http://localhost:8000` for the frontend and `http://localhost:8000/docs` for the API docs.

## Running Tests
change to week4 directory
```bash
PYTHONPATH=. poetry run pytest -q backend/tests
```

## Running Coverage
change to week4 directory
```bash
PYTHONPATH=. poetry run coverage run -m pytest -q backend/tests
```

## Style and Safety Guardrails
- **Tooling**: Uses `black` for formatting and `ruff` for linting
- **Safe commands to run**:
  - `make test` - runs pytest
  - `make format` - formats code safely with --fix
  - `make lint` - checks code without modifications
  - `make run` - starts the dev server
  - `make seed` - seeds the database
- **Commands to avoid**:
  - `git reset --hard` - discards all local changes
  - `git clean -fd` - removes untracked files
  - `rm -rf` on project directories
- **Lint/test gates**: Always run `make lint` and `make test` before committing

## Code Navigation
- **Entry point**: `backend/app/main.py` - FastAPI app creation and startup events
- **Routers**: `backend/app/routers/` - API route handlers (notes, action items)
- **Tests**: `backend/tests/` - Test files (conftest.py has fixtures)
- **Database**: SQLite at `backend/app.db` (created on first run)
- **DB seeding**: Run `make seed` or `poetry run python -m backend.app.seed` to populate initial data

## Workflow Snippets
- **When asked to add an endpoint**: First write a failing test, then implement
- **When fixing a bug**: Write a test that reproduces the bug before fixing
- **Refactoring**: Run tests before and after to ensure nothing breaks
- **Adding a new module**: Create in `backend/app/services/` and add corresponding test in `backend/tests/`
